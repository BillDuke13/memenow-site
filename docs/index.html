memenow-site · Documentation

One studio, one continuous narrative.

This site is the editorial home of the memenow studio. It is a SvelteKit application rendered on Cloudflare Workers, designed so each route reads as a chapter of a single long-form piece rather than as an independent page in a marketing tree.

What lives here

These pages are the human-readable companion to the source. The Markdown files at the repo root (README.md, CLAUDE.md, AGENTS.md) and the skill files under .claude/ are written for tooling and contributors; the HTML files here describe how the system is shaped and why.

If you are coming to the codebase fresh, read the documents in this order:

  1. Architecture — the moving parts and how they connect.
  2. Routes & content — every public URL, the chapter it represents, and where its copy lives.
  3. Components — the catalog of brand, layout, home, and UI primitives.
  4. Styling & tokens — design tokens, the .c-* / .l-* namespaces, and the world-layer overlay.
  5. Motion & reveal — Lenis, the scroll-driven reveals, and the rules for honoring prefers-reduced-motion.
  6. Deployment — the Cloudflare Workers pipeline, the verification gate, and how to roll back.
  7. Brand assets — the logo system and the script that regenerates every variant.

House rules at a glance

  • Svelte 5 runes only. The compiler is in forced runes mode for everything outside node_modules. Use $state, $props, $derived, $effect; never $: or export let.
  • Tailwind CSS v4 in CSS. Plugins are declared in src/routes/layout.css; there is no tailwind.config.{js,ts}. Class lists are sorted by prettier-plugin-tailwindcss — do not hand-order them.
  • Components are .c-* primitives. The visual system is built from BEM-style component classes that consume tokens, not from raw Tailwind utility soup.
  • Reduced motion is honored everywhere. Every animation utility — Lenis, reveal, magnetic, the matching CSS — has a prefers-reduced-motion: reduce branch.
  • pnpm only. engine-strict=true is set, and npm/yarn lockfiles must never appear.
  • Pre-push gate. pnpm lint && pnpm check && pnpm test, or invoke the /verify Claude skill.
Updating the docs When you change a route, component, or pipeline, update the matching HTML page in this directory. The Markdown files at the repo root cover the same surface for tooling — keep them in sync, but write the long-form prose here.
