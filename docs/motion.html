Motion & reveal

Smooth scroll, scroll-triggered reveals, and a magnetic hover. Three primitives, three tests, and a single non-negotiable rule: prefers-reduced-motion: reduce turns every one of them off.

Smooth scroll — Lenis

startLenis() in src/lib/utils/motion.ts dynamically imports the lenis package and starts a single instance, driving its raf loop until stopLenis() is called. The root layout boots it on mount and tears it down on destroy: 

// src/routes/+layout.svelte
import { startLenis, stopLenis } from '$lib/utils/motion';

onMount(() => {
    startLenis();
    return () => stopLenis();
});

The lifecycle is server-safe: startLenis early-returns when window is undefined, when reduced motion is requested, or when an instance is already running. The dynamic import keeps Lenis off the SSR critical path.

Scroll-triggered reveals — reveal action

The reveal Svelte action attaches an IntersectionObserver and flips data-revealed="true" on the element when it crosses the threshold. CSS rules in components.css animate from there.

Reveal targets

Pattern Markup CSS
Word-by-word title use:reveal={{ splitWords: true }} .reveal-word + .reveal-word-inner (translate-Y stagger via the --i custom property)

Word-splitting safety

When splitWords: true is passed, the action rebuilds the element's content word-by-word. It uses DOM nodes (not innerHTML) so any < or > characters in the source text are treated as inert text. The regression test in motion.svelte.test.ts covers an XSS attempt (<img src=x onerror=alert(1)>) and verifies it never becomes markup.

Children are preserved when not splitting

The non-splitting path leaves the existing element subtree intact and only attaches the observer. This is what makes use:reveal safe on rich content (em, strong, etc.).

Magnetic hover — magnetic action

use:magnetic={strength} nudges an element toward the cursor by a fraction of the offset, with an inertial release. Used by ArtifactCta with strength = 0.22. strength === 0 or reduced motion makes the action a no-op so callers can pass the strength conditionally without branching the template. 

<a href={href} use:magnetic={magnet ? 0.22 : 0} class="c-artifact-cta">
    {@render children()}
</a>

Reduced motion

Every motion primitive checks prefers-reduced-motion: reduce and degrades gracefully:

  • startLenis never instantiates Lenis — native browser scroll stays in effect.
  • reveal sets data-revealed="true" immediately and skips the word-splitting transform; existing children remain untouched.
  • magnetic returns the no-op cleanup form — no event listeners attached, no transforms applied.
  • tokens.css contains a global override: animation-duration, transition-duration, and scroll-behavior all collapse to near-zero.
  • components.css nukes the reveal transforms and clip paths under the same media query so anything we missed still ends up visible.
House rule If you add a new motion primitive, add a reduced-motion branch on the same commit. It is reviewer-blocking; the existing tests demonstrate the pattern.

Page-level transitions

The site does not use a route transition library. The continuous scene is created by the world-layer overlay (world-layer.css), the per-route fog (scene.css), and the route-family attribute swap on <html> — none of which require remounting content.
