Motion & reveal

Smooth scroll, scroll-triggered reveals, a magnetic hover, and cross-page View Transitions. A single non-negotiable rule runs through all of them: reduced motion — from the OS setting or the in-page toggle — turns every one of them off.

Smooth scroll — Lenis

startLenis() in src/lib/utils/motion.ts dynamically imports the lenis package and starts a single instance, driving its raf loop until stopLenis() is called. The root layout boots it on mount and tears it down on destroy: 

// src/routes/+layout.svelte
import { startLenis, stopLenis } from '$lib/utils/motion';

onMount(() => {
    startLenis();
    return () => stopLenis();
});

The lifecycle is server-safe: startLenis early-returns when window is undefined, when reduced motion is requested, or when an instance is already running. The dynamic import keeps Lenis off the SSR critical path.

Two lighter-weight exports — pauseLenis() and resumeLenis() — stop and restart the existing instance without tearing it down. The root layout uses them to bracket cross-page View Transitions: Lenis is paused before document.startViewTransition so its rAF loop can't shift the old/new viewport snapshots, then resumed when the transition finishes. Both are no-ops when Lenis never started (reduced motion or SSR).

Scroll-triggered reveals — reveal action

The reveal Svelte action attaches an IntersectionObserver and flips data-revealed="true" on the element when it crosses the threshold. CSS rules in components.css animate from there.

Reveal targets

Pattern Markup CSS
Word-by-word title use:reveal={{ splitWords: true }} .reveal-word + .reveal-word-inner (translate-Y stagger via the --i custom property)
Soft fade-in class="reveal-fade" use:reveal .reveal-fade (opacity + 20px translate)
Image clip-reveal class="reveal-clip" use:reveal .reveal-clip (clip-path inset from bottom)

Word-splitting safety

When splitWords: true is passed, the action rebuilds the element's content word-by-word. It uses DOM nodes (not innerHTML) so any < or > characters in the source text are treated as inert text. The regression test in motion.svelte.test.ts covers an XSS attempt (<img src=x onerror=alert(1)>) and verifies it never becomes markup.

Children are preserved when not splitting

The non-splitting path leaves the existing element subtree intact and only attaches the observer. This is what makes use:reveal safe on rich content (em, strong, etc.).

restDuringViewTransition

Pass restDuringViewTransition: true on any element that is a View Transition morph target. During a cross-page transition, document.startViewTransition captures a before/after snapshot of the element's rendered state. If that element is still in its pre-reveal clipped state, the morph animates toward an empty box rather than real content. With the flag set, the action reads the module-scoped viewTransitionUpdating boolean (written by setViewTransitionUpdating in motion.ts) and, if a transition is in flight, presents the element at rest immediately instead of starting an entrance animation. EditorialHero passes this flag on its title.

Magnetic hover — magnetic action

use:magnetic={strength} nudges an element toward the cursor by a fraction of the offset, with an inertial release. Used by ArtifactCta with strength = 0.22. strength === 0 or reduced motion makes the action a no-op so callers can pass the strength conditionally without branching the template. 

<a href={href} use:magnetic={magnet ? 0.22 : 0} class="c-artifact-cta">
    {@render children()}
</a>

Scroll-driven world drift & reading progress

The root layout (src/routes/+layout.svelte) runs a single requestAnimationFrame-throttled scroll handler that reads document.documentElement.scrollTop — so it works with Lenis or with the native scroll that reduced-motion users get — and drives two things on the same tick:

  • It writes the scroll fraction (0–1) to --world-scroll on <html>. world-layer.css reads it to translate and scale .world-layer__taichi, giving the persistent underlay a slow parallax drift.
  • It sets the width of the .c-progress__bar reading-progress hairline, which is aria-hidden wayfinding rather than an announced status.

Reduced motion

Reduced motion is governed by two parallel triggers, both of which collapse all motion primitives to no-ops. The single JS gate is prefersReducedMotion() in motion.ts:

  1. OS query — prefers-reduced-motion: reduce. In CSS this is guarded by :root:not([data-motion]) so it only applies before the boot script runs or when scripting is off. In JS, prefersReducedMotion() falls back to window.matchMedia when data-motion is absent.
  2. In-page toggle — MotionSwitch.svelte in the floating dock writes data-motion="full" or data-motion="reduced" on <html> and persists the choice to localStorage['memenow.motion']. The app.html boot script reads that key and writes the attribute before first paint so the preference applies immediately. When the attribute is present it is the single source of truth in both CSS (:root[data-motion='reduced']) and JS — it can both honor and override the OS setting.

When motion is reduced, every primitive degrades gracefully:

  • startLenis never instantiates Lenis — native browser scroll stays in effect. MotionSwitch calls stopLenis() / startLenis() directly when the preference changes after hydration.
  • reveal sets data-revealed="true" immediately and skips the word-splitting transform; existing children remain untouched.
  • magnetic returns the no-op cleanup form — no event listeners attached, no transforms applied.
  • tokens.css contains a global override: animation-duration, transition-duration, and scroll-behavior all collapse to near-zero.
  • components.css removes the reveal transforms and clip paths under both triggers so anything not caught by the JS gate still ends up visible.
  • world-layer.css zeroes the --world-scroll parallax transform, so the Tai Chi underlay holds still.
  • Cross-page View Transitions are skipped entirely — onNavigate in +layout.svelte returns early instead of calling document.startViewTransition, producing an instant DOM swap with no animation.
House rule If you add a new motion primitive, add a reduced-motion branch on the same commit. It is reviewer-blocking; the existing tests demonstrate the pattern.

Cross-page View Transitions

Same-document (SPA) navigations crossfade via the View Transitions API. The onNavigate hook in src/routes/+layout.svelte calls document.startViewTransition; browsers that do not support the API receive an instant DOM swap instead. The transition is skipped entirely under reduced motion (OS query or the in-page toggle) so no snapshot or animation is started.

Lenis is paused before the transition and resumed when it finishes (via transition.finished). Without the pause, Lenis's native-scroll rAF loop can shift the old and new viewport snapshots and cause a visible jump.

The hero title (.c-editorial-hero__title) carries view-transition-name: hero-title in components.css, giving it a shared-element morph between routes instead of dissolving with the page body. Because the morph must capture real text — not the pre-reveal clipped state — the reveal action skips its word-rise entrance on elements that pass restDuringViewTransition: true. The flag is read from the module-scoped viewTransitionUpdating boolean, set via setViewTransitionUpdating(true) before the transition and cleared when it finishes.

Page-level continuity

Beyond the View Transition crossfade, the site's continuous scene comes from the world-layer overlay (world-layer.css), the per-route fog (scene.css), and the route-family attribute swap on <html> — none of which require remounting content.
