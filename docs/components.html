Components

The Svelte component catalog grouped by responsibility. Every component is a Svelte 5 rune-mode module — props are declared with $props(), never with export let.

Brand

Component Path Purpose
MemenowMark lib/components/brand/MemenowMark.svelte The brand mark and lockups, served as a <picture> with avif / webp / png at @1x and @2x.

MemenowMark accepts a variant (mark, lockup-vertical, lockup-horizontal, reverse), a size (sm | md | lg | xl mapped to a height in pixels), and an optional eager flag to opt out of lazy loading. The image base URL is derived from the variant — never hand-write <img src="/brand/…">; use this component.

Home — narrative scroll primitives

Component Path Notes
EditorialHero lib/components/home/EditorialHero.svelte Full-viewport hero with optional background image, scrim, eyebrow row, large title, lede, and scroll cue. Title uses reveal with splitWords: true; lede and cue use reveal-fade.
Chapter lib/components/home/Chapter.svelte One narrative section. Supports align (center | left | split), banner (extends image to inner edge), and headless (drops the number / label row). Optional cta snippet renders below the body.

Chapter alignment recipes

  • align="center" (default) — title centered, body centered, single column. Best for openers and short statements.
  • align="left" — title and body left-aligned. Used for quieter chapters that don't carry a hero image.
  • align="split" — sticky title in the left column, body / image / CTA flow in the right. Used by the Yinyang and Quant chapters on / and /products.
  • banner — image extends to the chapter's inner edge with a 21:9 aspect ratio. Used by the Quant chapter on the home page.
  • headless — drops the 01 — Label row and keeps just the title. Used when adjacent chapters already establish the cadence (e.g. home The studio).

The Chapter component has dedicated browser tests in Chapter.svelte.test.ts covering each alignment and the headless behavior.

UI primitives

Component Path Notes
Button lib/components/ui/Button.svelte Native <button> with the .c-button primitive. variant: primary | secondary | ghost | link | on-scrim. size: sm | md | lg.

Class composition

cn(...inputs) in src/lib/utils/cn.ts is a thin clsx wrapper. There is no tailwind-merge: components in this codebase are .c-* primitives that do not stack conflicting Tailwind utilities. If you ever need to compose conflicting utilities, swap in tailwind-merge at that site rather than working around the limitation. 

import { cn } from '$lib/utils/cn';

<button class={cn('c-button', `c-button--${variant}`, size !== 'md' && `c-button--${size}`, className)}>
    {@render children()}
</button>

Snippets & children

Components that wrap content take a children: Snippet prop and call {@render children()}. Components with optional secondary slots (CTA, header) take a named cta or similar Snippet. There is no slot fallback API in runes mode; use a default Snippet argument instead.

Adding a new component Pick a directory by responsibility (brand, home, layout, ui). Add a matching .c-* primitive in src/lib/styles/components.css if it needs styling beyond what the existing primitives offer. If it has interactive behavior, add a *.svelte.test.ts alongside it so the browser project picks it up.
