Brand assets

The mark, lockups, and app icons all derive from a small set of PNG masters via scripts/build-brand-assets.py. Replace the master, run the script, commit the regenerated derivatives.

Inputs

Source PNGs live in static/brand/originals/. The script expects four files:

Master Used for
lockup-vertical.png Vertical lockup variants and the cropped square mark.
lockup-horizontal.png Horizontal lockup variants.
reverse-on-coral.png Reverse lockup on the coral background. The coral is preserved verbatim.
app-icon.png App icon raster, scaled down to PWA / favicon sizes.

Outputs

Running python3 scripts/build-brand-assets.py produces:

  • For each lockup and the square mark: PNG, WebP, and AVIF at @1x / @2x / @3x.
  • app-icon-{16,32,180,512}.png for browser tabs, manifest, and Apple touch icons.
  • static/favicon.ico as a multi-resolution (16 / 32 / 48) ICO.
  • static/brand/site.webmanifest referenced by app.html.

The mark is derived by cropping the top ~66% of the vertical lockup and trimming transparency. The parchment background of the source PNGs is replaced with full transparency using a per-channel fuzz of 12 (covers JPEG-style ringing and dithering).

Pipeline

python3 scripts/build-brand-assets.py

The script depends on Pillow (PIL). It is invoked manually whenever a master changes; there is no build hook. Always run it from the repo root so the derived paths land in the right place.

Where assets are referenced

Asset Consumer
brand/mark.{png,webp,avif} FloatingDock.svelte (24px in the dock pill); MemenowMark variant="mark".
brand/lockup-horizontal.{png,webp,avif} Footer.svelte via MemenowMark variant="lockup-horizontal".
brand/lockup-vertical.{png,webp,avif} Reserved for future hero / press treatments via MemenowMark variant="lockup-vertical".
brand/reverse-on-coral.{png,webp,avif} Reserved for dark / coral surfaces via MemenowMark variant="reverse".
brand/app-icon-{16,32,180,512}.png app.html link tags + brand/site.webmanifest.
favicon.ico app.html top-of-document <link rel="icon">.

The signature ornament

The coral pink diamond is the only ornament in the system. It is rendered as a CSS mask (no extra image asset) by SealMark.svelte and as a pseudo-element on chapter numbers in components.css. Do not substitute SVG, emoji, or Unicode glyphs.

Open Graph imagery

static/og/ holds the share images: og-default.jpeg wired into app.html, plus per-product overrides for og-yinyang.jpeg and og-quant.jpeg consumed by their respective +page.svelte heads. These are hand-authored, not generated.

Don't commit staging masters .brand-staging/ is gitignored — it holds chatgpt-create-image exploratory variants kept locally as an audit trail. Production masters live in static/brand/originals/.
