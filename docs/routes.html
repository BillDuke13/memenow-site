Products — /products and children

/products is the index. The two sub-routes are deeper readings.

  • /products/yinyang — narrative pitch for the Yinyang Agent. Two chapters (The reading, The discipline) and a CTA out to the product domain. Defines its own OG image (/og/og-yinyang.jpeg).
  • /products/quant — capability brief for the Quant Research practice. Opens with a coral-pink disclaimer card (Card variant="outlined" element="pink") that re-states this is not an offer and links to /disclaimer; followed by chapters for Crypto perpetuals and A-shares. Has its own OG image (/og/og-quant.jpeg).

Company — /company

Studio page. Three chapters: The studio, How we work, Get in touch. The hero image is /images/company.*.

Legal trio — /privacy, /terms, /disclaimer

All three carry <meta name="robots" content="noindex"> and have no hero image — the page renders against the legal fog only. They are still listed in the sitemap because the URLs are stable and we want them discoverable from the footer.

/privacy is intentionally short: no third-party analytics, no advertising, single browser-local key (memenow.theme) for the theme toggle, and Cloudflare's standard request metadata.

/terms covers scope (memenow.xyz only), no warranty, the explicit "no offer" clause, and Delaware governing law.

/disclaimer is the long form of the warning surfaced on /products/quant. It is linked from the Quant disclaimer card and from the footer.

Sitemap — /sitemap.xml

The sitemap is a +server.ts route with prerender = true so it ships as a static file inside the asset bundle. Its URL list lives in src/lib/data/sitemap.ts as a ReadonlyArray of [path, lastmod] pairs. Hand-bump the lastmod when content meaningfully changes — search crawlers discount uniform "everything updated today" stamps. 

// src/lib/data/sitemap.ts
export const SITEMAP_ROUTES: ReadonlyArray<readonly [path: string, lastmod: string]> = [
    ['/', '2026-05-06'],
    ['/products', '2026-05-06'],
    // …
];

The handler in +server.ts emits a one-hour Cache-Control. Tests in server.test.ts verify the content type, the cache header, and that every URL appears exactly once paired with its frozen lastmod.

Adding a new route

  1. Create src/routes/<path>/+page.svelte. Compose with EditorialHero + Chapter; provide <svelte:head> with title, description, and (if it deserves social sharing) og:title / og:description / og:image.
  2. Decide the route family. If the URL belongs in the legal trio, add it to LEGAL_PREFIXES in src/lib/utils/route-family.ts and add a test case.
  3. If you want it in the floating dock or footer, add it to src/lib/data/navigation.ts.
  4. Append the path and a YYYY-MM-DD lastmod to src/lib/data/sitemap.ts. Update the lastmod whenever the page's content meaningfully changes.
  5. If the page should be hidden from indexing, add <meta name="robots" content="noindex"> in its <svelte:head>.
  6. Run pnpm lint && pnpm check && pnpm test before pushing.
Editorial language All copy on the public site uses standard American English. Keep chapter titles short (typically <= 18 characters per visual line) and write ledes at human-thoughtful cadence rather than marketing copy.
