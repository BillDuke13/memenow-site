Architecture

memenow-site is a SvelteKit 2 application running on a Cloudflare Worker. It serves SSR HTML for first paint, hydrates as an SPA, and treats the entire site as a single continuous editorial scene.

The moving parts

Layer Responsibility Where it lives
Worker entrypoint Serves SSR HTML, hydrates client, serves static assets via the ASSETS binding. .svelte-kit/cloudflare/_worker.js (built); wrangler.jsonc
SSR hooks Resolves the route family and rewrites the data-route-family placeholder in the HTML stream so per-route fog is correct on first paint. src/hooks.server.ts
Document shell Inline boot script picks up the saved theme, OG / Twitter / icon meta, manifest, route-family placeholder. src/app.html
Root layout Imports the Tailwind v4 entrypoint, boots Lenis on mount, mirrors the route family on SPA navigations. src/routes/+layout.svelte
Routes One +page.svelte per public URL plus the prerendered /sitemap.xml. src/routes/**
Component library Brand, layout, home, and UI primitives consumed by the routes. src/lib/components/
Data sources Single source of truth for navigation, products, and the sitemap URL list. src/lib/data/
Styling Design tokens, scene fog, layout utilities, component primitives, world-layer overlay. src/lib/styles/
Utilities Class composition (cn), motion (startLenis, reveal, magnetic), route classification. src/lib/utils/

Request lifecycle

A first request to /products/yinyang walks the system as follows:

  1. Cloudflare Workers invokes _worker.js. The SvelteKit adapter hands the request to the platform handler.
  2. hooks.server.ts calls resolveRouteFamily('/products/yinyang'), gets 'threshold', and rewrites data-route-family="pending" in the HTML stream to data-route-family="threshold". The CSS selectors in scene.css can now activate the correct fog without waiting for hydration.
  3. SvelteKit renders +page.svelte on the Worker (no +page.server.ts for this route, so there is no extra load step) and streams the full document. Only /sitemap.xml is prerendered; every other route is server-rendered on each request.
  4. The browser receives HTML, runs the inline theme boot script in app.html to apply the saved or system theme to data-theme, and begins parsing.
  5. Hydration runs +layout.svelte, which boots Lenis (skipped under reduced motion) and registers an $effect that re-runs resolveRouteFamily for SPA navigations.
  6. The page itself renders EditorialHero and a stack of Chapters. Each one has a reveal action attached to its title and lede; an IntersectionObserver flips data-revealed="true" on each element as it scrolls into view.

SSR and SPA share one classifier

Two pieces of code consume resolveRouteFamily:

  • src/hooks.server.ts — runs on every full document load. Its transformPageChunk rewrites the placeholder in the HTML stream.
  • src/routes/+layout.svelte — an $effect reading page.url.pathname updates document.documentElement after client-side navigations (where hooks do not run).

Both call the same exported function so the result cannot drift. The unit tests in src/lib/utils/route-family.test.ts guard the classification rules — including the deliberate equality + segment-boundary check that prevents /privacy-mode from being misclassified as legal.

SvelteKit configuration

  • Adapter: @sveltejs/adapter-cloudflare. The build emits to .svelte-kit/cloudflare/ and wrangler.jsonc points at it.
  • Forced runes mode. svelte.config.js sets compilerOptions.runes to true for every file outside node_modules. Library code is exempt.
  • TypeScript strict + checkJs. tsconfig.json extends the SvelteKit-generated config and adds strict, checkJs, rewriteRelativeImportExtensions, and the Cloudflare runtime types via types: ['./src/worker-configuration.d.ts', 'node'].
  • App.Platform typing. src/app.d.ts declares the platform as { env: Env; cf: CfProperties; ctx: ExecutionContext; } backed by the generated runtime types.

Worker configuration

  • compatibility_date: "2026-05-06", compatibility_flags: ["nodejs_compat", "nodejs_als"].
  • assets.binding: "ASSETS" with directory: ".svelte-kit/cloudflare" — the SvelteKit adapter handles asset wiring.
  • observability.enabled: true and upload_source_maps: true are on so production tracebacks resolve to source.
  • No bindings are declared today (no KV, D1, R2, Durable Objects, services). Add new bindings to wrangler.jsonc and run pnpm cf-typegen to refresh src/worker-configuration.d.ts.

Test architecture

vite.config.ts defines two Vitest projects:

  • client — uses @vitest/browser-playwright with headless Chromium. Includes only src/**/*.svelte.{test,spec}.{js,ts}; excludes src/lib/server/**.
  • server — Node environment for everything else under src/**/*.{test,spec}.{js,ts}.

expect.requireAssertions is on, so a passing test that contains no assertions fails. Run all of them once with pnpm test; use pnpm test:unit only in interactive sessions.

Don't edit the worker types by hand src/worker-configuration.d.ts is regenerated whenever wrangler.jsonc bindings change. Hand edits will be lost the next time pnpm cf-typegen runs. The file is excluded from Prettier and ESLint for the same reason.
