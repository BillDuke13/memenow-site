Styling & tokens

A two-tone palette (sage green + coral pink + warm white), one display family (Manrope) and one body family (Inter), and a strict .c-* / .l-* namespacing convention. Every value comes from a CSS custom property — never a literal.

Stylesheet hierarchy

src/routes/layout.css is the only file Tailwind sees. It imports the framework, declares the v4 plugins, and pulls the rest of the stylesheets: 

@import 'tailwindcss';
@plugin '@tailwindcss/forms';
@plugin '@tailwindcss/typography';

@import '../lib/styles/tokens.css';
@import '../lib/styles/scene.css';
@import '../lib/styles/layout.css';
@import '../lib/styles/components.css';
@import '../lib/styles/world-layer.css';

@custom-variant dark (&:where([data-theme='dark'], [data-theme='dark'] *));

The order matters. Tokens load first, then scene fog (so it can read tokens), then layout utilities, then component primitives, then the world-layer overlay. The dark custom variant lets Tailwind utilities respond to data-theme="dark" without relying on the OS-level prefers-color-scheme.

--brand-coral is an alias of --accent-pink reserved for the logo's adjacent signature elements; do not introduce a third coral token.

Type

  • Display — Manrope Variable, with Inter as fallback. Used for hero and chapter titles, ledes, lockup typography. Letter-spacing tightens at display sizes (--ls-display: -0.04em).
  • Body — Inter, 400 / 500 / 600 weights. Used for everything that isn't a heading.
  • Mono — JetBrains Mono / IBM Plex Mono / system mono. Used for eyebrow rows, the dock chips, and inline code.
  • Type sizes use fluid clamp() ramps: --type-display-xl down to --type-overline. Reach for the token, not a hard rem value.

Spacing

A 4-pixel modular scale: --spacing-1 (0.25rem) through --spacing-32 (8rem). Use the token, not a hand-tuned padding.

Radius, shadow, motion

  • Radius — keep small. --radius-sm (4px) is the default, --radius-md (8px) for cards and media, --radius-lg (12px) only when something genuinely demands it. --radius-pill for the floating dock and tags.
  • Shadow — warm grey. --shadow-sm through --shadow-xl. Dark theme overrides them with proper black.
  • Motion — --motion-duration-fast / -base / -slow = 150ms / 250ms / 400ms. --motion-ease-standard for symmetric easing, --motion-ease-out for entrances. Reduced motion strips them.

Namespaces

Namespace Where Purpose
.l-* lib/styles/layout.css Layout utilities. .l-content, .l-stack, .l-grid, .l-section, .l-divider, .l-sr-only. Routes and pages own composition; these are the frames they hang composition on.
.c-* lib/styles/components.css Component primitives. Every Svelte component in lib/components/ that has visual style maps to a single .c-* root with optional --variant modifiers (.c-button--primary, .c-chapter--align-split, …). Variants and elements use BEM modifiers, never extra classes from another namespace.
.world-layer / .c-route-scene lib/styles/world-layer.css The fixed underlay (.world-layer) and the hero scene container (.c-route-scene). The world layer reads --scene-fog-active from scene.css so navigating between routes shifts the fog without remounting anything.
data-revealed / .reveal-* lib/styles/components.css Wired to the reveal Svelte action in lib/utils/motion.ts. .reveal-fade, .reveal-clip, and the .reveal-word / .reveal-word-inner pair for word-by-word splits.

Per-route fog

scene.css defines two named fog stacks (--scene-fog-celestial, --scene-fog-legal) and selects between them based on html[data-route-family]: 

:root {
    --scene-fog-active: var(--scene-fog-celestial);
}
[data-route-family='legal']     { --scene-fog-active: var(--scene-fog-legal); }
[data-route-family='threshold'] { --scene-fog-active: var(--scene-fog-celestial); }

The world-layer overlay reads --scene-fog-active, so swapping the family attribute during navigation is enough to retint the underlay without redrawing the DOM. Hero scrims (--scene-scrim-soft, --scene-scrim-strong) live in the same file and are consumed by RouteScene and EditorialHero.

Theming

Light is the default. data-theme="dark" on <html> re-binds every token at the document root. The app.html boot script selects between user choice (localStorage.memenow.theme) and system preference before SvelteKit hydrates so there is no light-flash on dark systems. ThemeSwitch.svelte writes both the attribute and the storage key when the user toggles.

No raw color literals If you find yourself writing color: #4a7c59 in a component, it should probably be color: var(--accent-green). The token is what survives a future palette tweak.

Tailwind v4 specifics

  • No config file. Plugins are declared with @plugin '...' inside src/routes/layout.css. Adding a Tailwind plugin means adding the devDependency and the @plugin line.
  • Class lists are sorted by Prettier. prettier-plugin-tailwindcss reads tailwindStylesheet: './src/routes/layout.css' from .prettierrc and orders utility classes deterministically. Do not hand-order them.
  • Components first, utilities second. The site is built from .c-* primitives consuming tokens. Tailwind utilities are reserved for ad-hoc spacing or flex tweaks inside a route, not for replicating component styles.
