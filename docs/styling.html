Styling & tokens

A two-tone palette (sage green + coral pink + warm white), one display family (Newsreader) and one body family (Inter), and a strict .c-* / .l-* namespacing convention. Every value comes from a CSS custom property — never a literal.

Stylesheet hierarchy

src/routes/layout.css is the only file Tailwind sees. It imports the framework and pulls the rest of the stylesheets: 

@import 'tailwindcss';

@import '../lib/styles/tokens.css';
@import '../lib/styles/scene.css';
@import '../lib/styles/layout.css';
@import '../lib/styles/components.css';
@import '../lib/styles/world-layer.css';

@custom-variant dark (&:where([data-theme='dark'], [data-theme='dark'] *));

The order matters. Tokens load first, then scene fog (so it can read tokens), then layout utilities, then component primitives, then the world-layer overlay. The dark custom variant lets Tailwind utilities respond to data-theme="dark" without relying on the OS-level prefers-color-scheme.

--brand-coral is an alias of --accent-pink reserved for the logo's adjacent signature elements; do not introduce a third coral token.

All palette values are authored in OKLCH (a perceptually uniform color space that gives even lightness steps and predictable color-mix). The sRGB hex values visible in the table above are approximations for reference; the source of truth is the oklch() literal in tokens.css. Every contrast ratio has been verified against the OKLCH values.

On Display-P3 screens, a wide-gamut accent layer pushes --accent-green and --accent-pink (and their -light variants) past the sRGB chroma ceiling for a richer brand presence. The overrides are declared in a @media (color-gamut: p3) block in tokens.css; they keep the exact lightness and hue of the sRGB fallback, so every contrast ratio is preserved — only chroma grows. Non-P3 engines never see the override.

Type

  • Display — Newsreader Variable (variable font, opsz axis), with system-serif fallbacks (Georgia, Times New Roman). Used for hero and chapter titles, ledes, and lockup typography. The optical-size axis is activated by font-optical-sizing: auto on html, so the typeface automatically uses finer, higher-contrast forms at large display sizes. Letter-spacing tightens at display sizes (--ls-display: -0.04em).
  • Body — Inter Variable (variable font, opsz axis), with system-sans fallbacks. Used for everything that isn't a display heading. Like Newsreader, the optical-size axis yields sturdier, more readable forms at small text sizes.
  • Mono — JetBrains Mono / IBM Plex Mono / system mono. Used for eyebrow rows, the dock chips, and inline code.
  • Type sizes use fluid clamp() ramps: --type-display-xl down to --type-overline. Reach for the token, not a hard rem value.

Spacing

A 4-pixel modular scale: --spacing-1 (0.25rem) through --spacing-32 (8rem). Use the token, not a hand-tuned padding.

Radius, shadow, motion

  • Radius — keep small. --radius-sm (4px) is the default, --radius-md (8px) for cards and media, --radius-lg (12px) only when something genuinely demands it. --radius-pill for the floating dock and tags.
  • Shadow — warm grey. --shadow-sm through --shadow-xl. Dark theme overrides them with proper black.
  • Motion — --motion-duration-fast / -base / -slow = 150ms / 250ms / 400ms. --motion-ease-standard for symmetric easing, --motion-ease-out for entrances. Reduced motion strips them.

Namespaces

Namespace Where Purpose
.l-* lib/styles/layout.css Layout utilities. .l-content, .l-stack, .l-grid, .l-section, .l-divider, .l-sr-only. Routes and pages own composition; these are the frames they hang composition on.
.c-* lib/styles/components.css Component primitives. Every Svelte component in lib/components/ that has visual style maps to a single .c-* root with optional --variant modifiers (.c-artifact-cta--ritual, .c-chapter--align-split, …). Variants and elements use BEM modifiers, never extra classes from another namespace.
.world-layer / .world-layer__taichi / .world-layer__grain lib/styles/world-layer.css The fixed underlay that renders behind every route. .world-layer paints the active fog, .world-layer__taichi the Tai Chi geometry with scroll-driven parallax drift, and .world-layer__grain a faint static film grain (inline SVG feTurbulence) tiled over the underlay for texture. The grain is hidden under prefers-reduced-transparency: reduce and forced-colors: active. The layer reads --scene-fog-active from scene.css so navigating between routes shifts the fog without remounting anything. Also declares the cross-page View Transition styles (::view-transition-old/new(root) crossfade).
data-revealed / .reveal-* lib/styles/components.css Wired to the reveal Svelte action in lib/utils/motion.ts. .reveal-fade, .reveal-clip, and the .reveal-word / .reveal-word-inner pair for word-by-word splits.

Per-route fog

scene.css defines two named fog stacks (--scene-fog-celestial, --scene-fog-legal) and selects between them based on html[data-route-family]: 

:root {
    --scene-fog-active: var(--scene-fog-celestial);
}
[data-route-family='legal']     { --scene-fog-active: var(--scene-fog-legal); }
[data-route-family='threshold'] { --scene-fog-active: var(--scene-fog-celestial); }

The world-layer overlay reads --scene-fog-active, so swapping the family attribute during navigation is enough to retint the underlay without redrawing the DOM. Hero scrims (--scene-scrim-soft, --scene-scrim-strong) live in the same file and are consumed by EditorialHero.

Theming

Light is the default. data-theme="dark" on <html> re-binds every token at the document root. The app.html boot script selects between user choice (localStorage.memenow.theme) and system preference before SvelteKit hydrates so there is no light-flash on dark systems. ThemeSwitch.svelte writes both the attribute and the storage key when the user toggles.

In-page motion toggle (data-motion)

MotionSwitch.svelte in the floating dock lets users override the OS prefers-reduced-motion setting. It writes data-motion="full" or data-motion="reduced" on <html>, persists the choice to localStorage['memenow.motion'], and starts or stops Lenis accordingly.

The attribute is the single runtime source of truth once scripting is active: when data-motion is present it overrides the OS query in both CSS and JS. The deliberate design is that <html> ships no static data-motion in the SSR HTML, so the CSS :root:not([data-motion]) selectors can fall back to the OS media query when scripting is off. The app.html boot script writes the attribute before first paint on all subsequent visits where a stored preference exists.

The matching CSS resets in tokens.css, components.css, world-layer.css, and layout.css respond to both triggers — :root:not([data-motion]) guards the OS query fallback; :root[data-motion='reduced'] governs the toggle. The JS gate in motion.ts (prefersReducedMotion()) mirrors this precedence: the attribute wins whenever present, falling back to window.matchMedia otherwise.

No raw color literals If you find yourself writing color: #4a7c59 in a component, it should probably be color: var(--accent-green). The token is what survives a future palette tweak.

Tailwind v4 specifics

  • No config file, no plugins. Configuration lives in src/routes/layout.css — the @import 'tailwindcss' entrypoint and the dark custom variant. No Tailwind plugins are registered today; if one is ever needed, add the devDependency and a @plugin '...' line there.
  • Class lists are sorted by Prettier. prettier-plugin-tailwindcss reads tailwindStylesheet: './src/routes/layout.css' from .prettierrc and orders utility classes deterministically. Do not hand-order them.
  • Components first, utilities second. The site is built from .c-* primitives consuming tokens. Tailwind utilities are reserved for ad-hoc spacing or flex tweaks inside a route, not for replicating component styles.
