Deployment

The site builds with the SvelteKit Cloudflare adapter and ships to a single Cloudflare Worker via Wrangler. Production is gated by the verification pipeline.

Authentication

The deploying user needs a Wrangler session against the Cloudflare account that owns the memenow-site Worker. wrangler login must be run interactively in the terminal — it cannot be automated from a coding agent. 

wrangler whoami
# if not logged in:
wrangler login

Pre-deploy verification

Run the verification pipeline before every deploy. The /verify Claude skill wraps it; manually it is: 

pnpm lint        # prettier --check && eslint
pnpm check       # svelte-kit sync && svelte-check
pnpm test        # vitest --run (client + server projects)

If lint fails, pnpm format auto-fixes most issues. If a browser test cannot launch Chromium, install it once with pnpm exec playwright install chromium.

Local Worker preview

When the change touches src/hooks.server.ts, a +server.ts route, or wrangler.jsonc, smoke-test the production build locally before deploying: 

pnpm preview

This runs pnpm build and then wrangler dev against the built Worker. Hit the routes that changed; watch the Wrangler output for compatibility, binding, or asset errors.

Deploy

pnpm deploy

This runs pnpm build followed by wrangler deploy. The deploy URL appears in the Wrangler output; capture it and confirm a 200 from /, /sitemap.xml, and the route that motivated the deploy.

Confirm before deploying The /deploy-cf Claude skill is human-invocation only (disable-model-invocation: true) — agents cannot trigger it autonomously. Always confirm with the user before running pnpm deploy.

Rollback

If a deploy regresses production, roll back via:

wrangler rollback

or via the Cloudflare dashboard: Workers & Pages → memenow-site → Deployments → Rollback.

Adding a binding

  1. Declare the binding (KV, D1, R2, Queue, Service, Durable Object) in wrangler.jsonc under the documented section.
  2. Run pnpm cf-typegen to regenerate src/worker-configuration.d.ts. Commit the regenerated file alongside the wrangler.jsonc change.
  3. Use the binding from App.Platform via the event.platform.env handle in server hooks or +server.ts routes. The platform shape is declared in src/app.d.ts.
  4. If the binding holds secrets (API keys, tokens), use wrangler secret put NAME. Never commit .dev.vars.
  5. Run pnpm preview with the binding present locally before deploying.

Hidden files and side artifacts

  • .wrangler/ — local Wrangler state (cache, sessions). Gitignored.
  • .svelte-kit/ — SvelteKit build output. Gitignored; rewritten by svelte-kit sync and the build.
  • static/.assetsignore excludes _worker.js and _routes.json so the Cloudflare asset bundle does not double-package the Worker entry alongside the static assets.

Environments

Today there is a single environment (production). To add a staging environment, introduce an [env.staging] block in wrangler.jsonc and deploy with wrangler deploy --env staging. The /deploy-cf skill accepts an environment hint as its argument.
