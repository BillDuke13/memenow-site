# memenow-site

[![CI](https://github.com/memenow/memenow-site/actions/workflows/ci.yml/badge.svg)](https://github.com/memenow/memenow-site/actions/workflows/ci.yml)
[![Deploy](https://github.com/memenow/memenow-site/actions/workflows/deploy.yml/badge.svg)](https://github.com/memenow/memenow-site/actions/workflows/deploy.yml)
[![CodeQL](https://github.com/memenow/memenow-site/actions/workflows/codeql.yml/badge.svg)](https://github.com/memenow/memenow-site/actions/workflows/codeql.yml)

The official website of [memenow](https://memenow.xyz) — an editorial single-narrative
[SvelteKit](https://svelte.dev/) site deployed to
[Cloudflare Workers](https://developers.cloudflare.com/workers/) via
[`@sveltejs/adapter-cloudflare`](https://kit.svelte.dev/docs/adapter-cloudflare).

## Requirements

- Node.js (current LTS) — `engine-strict=true` is set in `.npmrc`.
- [pnpm](https://pnpm.io/) — the only supported package manager for this repo.
- (Optional, for deploys) [`wrangler`](https://developers.cloudflare.com/workers/wrangler/)
  authenticated against the target Cloudflare account.

## Getting started

```sh
pnpm install
pnpm dev
```

The first run of the browser test project also needs Chromium:

```sh
pnpm exec playwright install chromium
```

## Scripts

| Command           | Purpose                                                   |
| ----------------- | --------------------------------------------------------- |
| `pnpm dev`        | Start the Vite dev server with HMR.                       |
| `pnpm build`      | Produce a production build for Cloudflare Workers.        |
| `pnpm preview`    | Build, then run `wrangler dev` locally.                   |
| `pnpm check`      | Run TypeScript and Svelte diagnostics.                    |
| `pnpm lint`       | Run Prettier and ESLint in check mode.                    |
| `pnpm format`     | Auto-format the repository with Prettier.                 |
| `pnpm test`       | Run unit and component tests once with Vitest.            |
| `pnpm test:unit`  | Run Vitest in interactive watch mode.                     |
| `pnpm deploy`     | Build and deploy to Cloudflare Workers via Wrangler.      |
| `pnpm cf-typegen` | Regenerate `src/worker-configuration.d.ts` from bindings. |

The standard pre-push gate is `pnpm lint && pnpm check && pnpm test` (also wrapped by the
`/verify` Claude skill).

## Project structure

```
src/
├── app.d.ts                # SvelteKit `App.Platform` typed against Cloudflare Env / CfProperties / ExecutionContext
├── app.html                # Document shell + theme/route-family boot script
├── hooks.server.ts         # Sets `data-route-family` on <html> for SSR (legal vs. threshold)
├── worker-configuration.d.ts  # Generated by `wrangler types` — do not edit by hand
├── routes/                 # File-based routes; see the table above
│   ├── layout.css          # Tailwind v4 entrypoint (imports tokens, scene, layout, components, world-layer)
│   ├── +layout.svelte      # Lenis lifecycle + SPA fallback for `data-route-family`
│   ├── +page.svelte        # Home
│   └── sitemap.xml/        # `+server.ts` + colocated tests; prerendered
└── lib/
    ├── components/
    │   ├── brand/          # MemenowMark (multi-format <picture>)
    │   ├── home/           # EditorialHero, Chapter — narrative scroll primitives
    │   ├── layout/         # AppShell, Footer, FloatingDock, RouteScene
    │   └── ui/             # Button, ButtonLink, Card, Badge, SealMark, ThemeSwitch, ArtifactCta
    ├── data/               # navigation.ts, products.ts, sitemap.ts (single source of truth)
    ├── styles/             # tokens.css, scene.css, layout.css, components.css, world-layer.css
    └── utils/              # cn.ts (clsx wrapper), motion.ts (Lenis + reveal/magnetic), route-family.ts
static/
├── brand/                  # Logo system (multi-format @1x/@2x/@3x), site.webmanifest, app icons
├── images/                 # Hero and chapter imagery (avif/webp/png)
├── og/                     # Open Graph share images
├── favicon.ico
└── robots.txt
scripts/
└── build-brand-assets.py   # Derives every brand variant from PNG masters in static/brand/originals/
docs/                       # HTML documentation for humans (architecture, components, motion, deploy, brand)
```

`wrangler.jsonc` configures the Worker (declare any KV / D1 / R2 / Queues bindings here, then
run `pnpm cf-typegen` to refresh `src/worker-configuration.d.ts`).

## Stack notes

- **Svelte 5 with runes mode forced** outside `node_modules` — use `$state`, `$props`,
  `$derived`, `$effect`. Legacy reactive syntax (`$:`, `export let`) is not supported.
- **Tailwind CSS v4** — configured in CSS via `@import 'tailwindcss'` and `@plugin '...'` in
  `src/routes/layout.css`. There is no `tailwind.config.{js,ts}`.
- **CSS architecture** — design tokens (`tokens.css`) drive component primitives in the `.c-*`
  namespace (`components.css`) and layout utilities in the `.l-*` namespace (`layout.css`).
  The world-layer overlay (`world-layer.css`) plus per-route fog (`scene.css`) make the whole
  site read as one continuous narrative scene rather than discrete pages.
- **Motion** — [Lenis](https://github.com/darkroomengineering/lenis) drives smooth scroll;
  Svelte actions in `lib/utils/motion.ts` provide IntersectionObserver-driven `reveal` and a
  cursor-following `magnetic` action. Every helper honors
  `prefers-reduced-motion: reduce`.
- **Theming** — `app.html` ships an inline boot script that reads `localStorage.memenow.theme`
  (or system preference) and sets `data-theme` before SvelteKit hydrates, avoiding a
  light-flash. `ThemeSwitch.svelte` toggles and persists the choice.
- **Route-family fog** — `hooks.server.ts` rewrites `data-route-family="pending"` in
  `app.html` to `legal` (privacy/terms/disclaimer) or `threshold` (everything else) for SSR;
  the root layout's `$effect` mirrors that on SPA navigations.
- **Vitest** runs two projects defined in `vite.config.ts`: `client` (browser mode via
  `@vitest/browser-playwright`, headless Chromium) for `*.svelte.{test,spec}.{js,ts}`, and
  `server` (Node) for the rest.
- **Cloudflare Workers** — `compatibility_date` is `2026-05-06`, `compatibility_flags`
  includes `nodejs_compat` and `nodejs_als`, and `upload_source_maps` and `observability`
  are both on.

## Brand assets

`static/brand/originals/` holds the PNG masters. `scripts/build-brand-assets.py` derives the
full system — mark, lockup-vertical, lockup-horizontal, and reverse-on-coral at @1x/@2x/@3x
in PNG/WebP/AVIF, plus app icons and `favicon.ico`. Re-run after replacing a master:

```sh
python3 scripts/build-brand-assets.py
```

## Continuous integration and deployment

Every pull request runs the same gate locally enforced by `pnpm lint && pnpm check && pnpm test`,
plus CodeQL static analysis and a Lighthouse audit against the built Worker. Pushing to `main`
deploys to Cloudflare Workers automatically via `cloudflare/wrangler-action`.

| Workflow                           | Triggers                                                  |
| ---------------------------------- | --------------------------------------------------------- |
| `.github/workflows/ci.yml`         | `pull_request`, `push: main` — lint, check, test, build   |
| `.github/workflows/deploy.yml`     | `push: main`, `workflow_dispatch` — `wrangler deploy`     |
| `.github/workflows/codeql.yml`     | `pull_request`, `push: main`, weekly cron                 |
| `.github/workflows/lighthouse.yml` | `pull_request`, `workflow_dispatch` — LH against local DO |
| `.github/dependabot.yml`           | weekly — grouped `npm` (pnpm) and `github-actions` PRs    |

The deploy workflow consumes the `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID`
organization secrets — no repository-level secret configuration is required. See
[`docs/ci-cd.html`](./docs/ci-cd.html) for the full pipeline reference, including the
production environment, caching strategy, Lighthouse budgets, manual triggers, and the
rollback runbook.

## Documentation
